GitHub Cheat – Assets Folder & Images

How to create an assets folder and use your images.

On this page
  1. What is an assets folder?
  2. Option 1: Create the folder on GitHub.com
  3. Option 2: “Create new file” folder trick
  4. Option 3: Make the folder on your computer
  5. Using your images in HTML & CSS

1. What is an assets folder?

GitHub doesn’t really have “folders” as a separate thing. A folder exists only when there are files inside it. An assets folder is just a regular folder in your repository where you store images, icons, or other media.

Remember: You don’t see a “New folder” button in GitHub. Instead, you create a folder by saving a file with folder-name/your-file-name in its path.

2. Option 1 – Create an assets folder on GitHub.com

This is easiest if you’re working directly in the browser.

Open your repository

Go to the GitHub repo where you want to add your images.

Start an upload
  1. At the top of the file list, click Add file → Upload files.
  2. You’ll see an area that says something like “Drag files here to upload”.
Tell GitHub you want a folder
  1. Drag your image(s) from your computer into the upload area.
  2. Above the upload area you’ll see the file path (it looks like your-repo-name/).
  3. Click at the end of that path and type: assets/ so it becomes something like: your-repo-name/assets/your-image.png
Save your changes
  1. Scroll down to the “Commit changes” area.
  2. Optionally type a short message like Add assets folder.
  3. Click Commit changes.

✅ Your repo now has an assets folder with your image files inside.

3. Option 2 – Use the “Create new file” folder trick

This trick is nice if you want to create the folder first and upload images later.

Create a new file
  1. In your repo, click Add file → Create new file.
  2. In the file name box, type something like assets/example.txt.
Add placeholder content

In the big text box, you can type anything (even just placeholder). This file only exists to create the folder.

Commit the file
  1. Scroll down to the commit section.
  2. Click Commit new file.

Now you’ll see an assets folder in your repository. You can click into it and use Upload files to add your images directly into that folder. You can delete the example.txt later if you don’t need it.

4. Option 3 – Create assets locally and push

Use this approach if you’re working on your project on your own computer (VS Code, etc.) and pushing to GitHub with Git.

  1. On your computer, inside your project folder, create a new folder called: assets.
  2. Put your image files inside that folder (for example: logo.png, background.jpg).
  3. Open a terminal in your project folder and run:
git add assets/*
git commit -m "Add assets folder with images"
git push

When you refresh your repo on GitHub, the assets folder and your images will be there.

5. Using your images in HTML & CSS

Once your images live in assets/, the path to an image looks like: 

assets/your-image-name.png

Example: Using an image in HTML

<img src="assets/logo.png" alt="Site logo">

Example: Using an image in CSS

.hero {
  background-image: url("assets/background.jpg");
}
Tip: If an image isn’t showing up, double-check:
  • The file name spelling (including .png, .jpg, etc.).
  • That the file actually lives in the assets folder in your repo.
  • That the path in your code matches the folder structure exactly.
