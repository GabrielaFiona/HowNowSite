3. Fixing Existing Footers That Use .wrap

If a page already has HTML like this:

<footer class="footer">
  <div class="wrap">
    ...footer content...
  </div>
</footer>

And your CSS previously used a grid on .footer .wrap, you‚Äôll center it using flex instead.

3.1 New Flex-Based Centering

.footer .wrap {
  max-width: 700px;
  margin: 0 auto;
  display: flex;
  flex-direction: column;
  align-items: center;
  text-align: center;
  gap: 12px;
}
Important: Things to Remove or Override

Search your CSS for old layout rules that fight with the new flex layout and remove or comment them out:

  • Any .footer .wrap { display: grid; ... }
  • Media queries like: 
    @media (min-width: 840px) {
  .footer .wrap {
    grid-template-columns: 1fr 1fr;
    ...
  }
}
  • Any extra justify-content or text-align rules on .footer or .wrap that clash:
    • text-align: left; on .footer
    • margin-left/right hacks that push the footer off-center

Keep only one layout system per element: either grid or flex, not both.

Why this works well in embeds

The footer content is wrapped in a centered container with a max-width. Inside that, flex handles stacking vs side-by-side. The iframe in Google Sites just constrains the outer width ‚Äî the layout still behaves normally.

5. Things To Check / Clean Up When Reusing These Patterns

  1. In your CSS, search for any .footer, .footer-inner, .wrap, or footer rules. Remove duplicates that re-define layout (display, grid-template-columns, etc.).
  2. Keep only one "layout block" per element:
    • Either .footer-inner { display:flex; ... } or display:grid;, not both.
    • Remove older grid settings that were used just to make two columns.
  3. Make sure you only have one <span id="year"> per page and one matching JS snippet.
  4. If a page previously used a different footer HTML structure, update the HTML to match one of the patterns here, then remove any now-unused footer styles from your CSS.
  5. Test each page in:
    • Your browser at the GitHub Pages URL.
    • Inside Google Sites after embedding the URL (resize to see mobile vs desktop behavior).
